## Задачи на Java:

1. Напишите Java-программу, которая:
 + Принимает на вход: логин, пароль и имя хоста БД; 
 + Осуществляет подключение к БД (мы точно знаем, что это Postgresql); 
 + Считает количество записей в таблице dq_rule; 
 + Создаёт строку в таблице log – в таблице два поля: дата/время создания строки и результат пункта c;

2. Напишите пару Java-классов. Родительский и дочерний.
 + В родительском классе должен быть метод, который возвращает факториал любого числа, переданного как параметр метода; 
 + Дочерний переопределяет этот метод и возвращает всегда факториал 10и; 
 + Сделайте так, чтобы нельзя было создать объект от родительского класса – только от дочернего;

3. Добавьте к решению предыдущей задачи третий класс – такой, который позволит получать факториал любого числа без создания экземпляра класса (объекта);

4. Напишите Java-программу, которая принимает на вход текстовый файл с параметрами в форме «ключ-значение»;
 + Программа должна прочитать файл и выдать его содержимое в отсортированном по ключу виде;

5. Напишите Java-программу, которая читает текстовый файл, содержащий логин и пароль;
 + Шифрует пароль с использованием ключа, который также вшифровывается в результат шифрования; 
 + Создаёт новый файл с шифрованным паролем и удаляет старый файл с открытым паролем; 
 + Другой метод того же класса должен уметь расшифровать таким образом зашифрованный пароль из файла с шифрованным паролем и печатать расшифрованный пароль в консоль, исходный файл с расшифрованным паролем создавать не надо; 
 + Без внешне передаваемого ключа шифрования. Ключ шифрования должен генерироваться самой программой и становится частью шифра, но только так, чтобы его нельзя было легко вычленить.

## Задачи на Postgresql:

При решении заданий рекомендуется использовать синтаксис Postgresql.

1. Сколько строк и полей и почему вернет SQL запрос “select * from A,B,C”, если
   + в A содержит 3 строки, B - 4 строки, а C - 5 строк?
   + в A содержит 0 строк, B - 1 строка, а C - 2 строки?

2. Предложите варианты отбора уникальных значений поля “name” из таблицы “A” (в СУБД только эта одна таблица). 
Необходимо минимум 4 различных варианта (известно минимум 7 разных вариантов).

3. Напишите SQL, используя таблицу “A”, возвращающий ровно 100 строк со значениями чисел от 1 до 100 в каждой строке. Ограничения: 

    1) не использовать connect by; 
    2) не использовать union/union all; 
    3) в запросе можно использовать только таблицу “A”, в которой ровно одно поле “COL“ с текстовым значением ‘X’, и в таблице “A” содержится 7 строк.

4. Предложите SQL (без использования **PIVOT**) для получения клиентских атрибутов (
таблица: **ATTR**; поля: **CLIENT_ID**, **PARAM_NAME**, **PARAM_VALUE**; первичный ключ: ID, PARAM_NAME; 
всего различных параметров 100: от ‘p1’ до ‘p100’) в строку (то есть один клиент – одна строка). 
На выходе должно быть 101 поле: **CLIENT_ID, P_1, …, P_100**.

5. Есть таблица:
````
   create table t(col varchar2(3));
   insert into t(col) values('AAA');
   insert into t(col) values('BBB');
   insert into t(col) values(null);
   insert into t(col) values('BBB');
   insert into t(col) values('BBB');
   insert into t(col) values('CCC');
   commit;
````
   Удалите дубликаты из неё без промежуточных таблиц, одним запросом.

7. Есть таблица лога выполнения некой операции. Назовём её log.
   Поля таблицы:
   > oper_id – идентификатор операции

   > start_ts – дата и время начала операции с точностью до секунды

   > end_ts – дата и время окончания операции с точностью до секунды

   Операции могут пересекаться по времени. То есть, какие-то из них работают параллельно.
   В таблице лежат данные за несколько дней.
   Необходимо посчитать среднее количество одновременно выполнявшихся операций по часам.
   То есть на выходе получить примерно:
+ 08:00 – 0 операций 
+ 09:00 – 2 операции 
+ 10:00 – 50 операций 
+ 11:00 – 42 операции 
+ 12:00 – 84 операции 
+ 13:00 – 11 операций

   И .тд.
